# Быстрый запуск Асси

Асси — локальный ИИ-ассистент для поиска и ответов по загруженным учебным медицинским материалам.

Самый простой способ запуска — через Docker Compose.

## Что будет запущено

| Компонент | Адрес | Назначение |
|---|---|---|
| Web-интерфейс | `http://127.0.0.1:3000` | Загрузка материалов, поиск и вопросы |
| FastAPI | `http://127.0.0.1:8000` | Backend API |
| Swagger UI | `http://127.0.0.1:8000/docs` | Интерактивная документация API |
| Healthcheck | `http://127.0.0.1:8000/api/v1/health` | Проверка состояния сервисов |
| PostgreSQL | `localhost:5432` | Документы, задания индексации и feedback |
| Qdrant | `http://127.0.0.1:6333` | Векторный индекс фрагментов |

## Ресурсы

| Ресурс                               |                             Минимум |                 Нормально |                                            Комфортно |
| ------------------------------------ | ----------------------------------: | ------------------------: | ---------------------------------------------------: |
| Оперативная память компьютера        | 8 ГБ — вероятен swap и возможен OOM |                 **16 ГБ** |                                             24–32 ГБ |
| Память, доступная Docker             |                    6 ГБ — ненадёжно |              **10–12 ГБ** |                                                16 ГБ |
| Свободное место перед первой сборкой |                     20 ГБ — впритык |          **25–30 ГБ SSD** |                                               40+ ГБ |
| Постоянно занято после сборки        |                   примерно 18–22 ГБ |                         — |                                зависит от материалов |
| CPU                                  |                              4 ядра |              **6–8 ядер** |                                            8–12 ядер |

**GPU не требуется.**

Стандартный поиск использует только Dense retrieval. Cross-encoder reranker отключён по результатам retrieval-эксперимента и не скачивается при обычном запуске.

## 1. Подготовь компьютер

Нужны только:

- Docker Desktop с Docker Compose;
- Git — либо скачанный ZIP-архив репозитория.

Если на компьютере нет Docker Desktop или Git, ниже — инструкции для установки Docker и Git:

- [Установка на macOS](install_macos.md);
- [Установка на Windows](install_windows.md).

## 2. Скачай проект

### macOS — Terminal

```bash
git clone https://github.com/ssd2008/MLproject.git
cd MLproject
```

### Windows — PowerShell

```powershell
git clone https://github.com/ssd2008/MLproject.git
Set-Location MLproject
```

Если Git не установлен, скачай репозиторий через GitHub: **Code → Download ZIP**, распакуй архив и открой терминал в полученной папке.

## 3. Запусти приложение

Команда одинакова на macOS и Windows:

```bash
docker compose up --build
```

**Первый запуск будет долгим.** Docker:

1. скачает образы PostgreSQL и Qdrant;
2. соберёт backend и frontend;
3. установит Python-зависимости;
4. скачает embedding-модель и Whisper-модель;
5. пропустит reranker, поскольку `RERANKER_ENABLED=false`;
6. применит SQL-миграции;
7. запустит приложение.

Не закрывай Docker Desktop и терминал, пока сервисы не перейдут в рабочее состояние.

Проверить состояние в другом терминале:

```bash
docker compose ps
```

Посмотреть ключевые логи:

```bash
docker compose logs -f model-init api frontend
```

В стандартных логах `model-init` должна появиться строка:

```text
Skipping reranker model: RERANKER_ENABLED=false
```

## 4. Открой Асси

Открой в браузере:

```text
http://127.0.0.1:3000
```

## 5. Пользуйся

1. Загрузи PDF, видео, текст или URL и дождись статуса `Готов`.
2. Выполни поиск по содержимому.
3. Задай вопрос Асси.
4. Проверь цитаты, страницы PDF или тайм-коды видео.

По умолчанию поиск выполняется только через dense embeddings и Qdrant. В обычных ответах `rerank_score` равен `null`.

## Экспериментальное включение reranker

Reranker не рекомендуется для стандартного использования текущего MVP: на benchmark он ухудшил MRR и точность первого результата, а медианная задержка выросла с `0.297` до `5.525` секунды.

Для повторного эксперимента измени `docker-compose.yml` или `.env`:

```dotenv
RERANKER_ENABLED=true
```

Пересоздай `model-init` и API:

```bash
docker compose up --build model-init api
```

Затем в запросе `/search` или `/answer` передай:

```json
{
  "use_reranker": true
}
```

Если глобальный флаг выключен, поле запроса не включает reranking и приложение продолжает использовать Dense retrieval.

## Остановка и повторный запуск

Остановить приложение:

```bash
docker compose down
```

Запустить повторно без пересборки:

```bash
docker compose up
```

Запустить в фоне:

```bash
docker compose up -d
```

Посмотреть логи фонового запуска:

```bash
docker compose logs -f
```

## Полный сброс данных

```bash
docker compose down -v
```

> **Осторожно:** параметр `-v` удаляет Docker volumes — тома с PostgreSQL, Qdrant, загруженными файлами и кэшем моделей. После этого данные будут потеряны, а включённые модели придётся скачивать заново.
