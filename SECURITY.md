# Security Policy

## Scope

Асси — локальный учебный MVP. Конфигурация Docker Compose предназначена для запуска на одном компьютере и не является production deployment.

Проект не следует публиковать напрямую в интернете: в текущей версии отсутствуют authentication, authorization, пользовательская изоляция, rate limiting и ресурсные квоты.

## Supported version

Исправления безопасности принимаются для последней версии ветки `master`.

## Reporting a vulnerability

Не публикуйте сведения об уязвимости в открытом issue до её исправления. Свяжитесь с владельцем репозитория через приватный канал GitHub, указанный в профиле, и приложите:

- краткое описание проблемы;
- затронутый endpoint или компонент;
- минимальные шаги воспроизведения;
- возможные последствия;
- предложенное исправление, если оно известно.

Не прикладывайте реальные медицинские документы, персональные данные, `.env`, API-ключи, пароли и другие секреты.

## Local deployment rules

- Оставляйте опубликованные Docker-порты привязанными к `127.0.0.1`.
- Не используйте демонстрационные настройки Compose для внешнего сервера.
- Перед публикацией логов удаляйте персональные данные и значения переменных окружения.
- Загружайте только те материалы, которые разрешено хранить и обрабатывать локально.
- Ответы Асси предназначены для обучения и не заменяют клиническое решение врача.

## Dependency and secret checks

Перед публичным релизом рекомендуется запускать:

```bash
ruff check .
python -m pytest
npm --prefix frontend run build
docker compose config --quiet
```

Для проверки истории репозитория на случайно закоммиченные секреты используйте `gitleaks` или аналогичный scanner.
